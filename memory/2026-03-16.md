### 一、协议层异常 (7种)

#### 1. `fcnt-duplicate` - FCnt重复
| 属性 | 说明 |
|------|------|
| **触发条件** | every-3rd-uplink (每第3个上行帧) |
| **行为** | 重复使用相同的FCnt值发送数据包 |
| **网络反应** | ChirpStack应拒绝重复FCnt的包（重放攻击防护） |
| **检测方式** | 检查`data_up`日志中的`f_cnt`重复警告 |
| **实际场景** | 设备复位后FCnt未恢复、恶意重放攻击 |

#### 2. `fcnt-jump` - FCnt跳变
| 属性 | 说明 |
|------|------|
| **触发条件** | every-5th-uplink |
| **行为** | FCnt突然增加1000（从当前值跳跃） |
| **网络反应** | 可能被接受（如果跳跃后仍大于上次值）或触发告警 |
| **检测方式** | 监控FCnt不连续增长模式 |
| **实际场景** | 设备掉电后FCnt丢失、恶意伪造 |

#### 3. `mic-corrupt` - MIC损坏
| 属性 | 说明 |
|------|------|
| **触发条件** | every-2nd-uplink |
| **行为** | 对MIC字段进行4bit随机翻转 |
| **网络反应** | **MIC验证失败，数据包被丢弃** |
| **检测方式** | ChirpStack日志显示MIC错误 |
| **实际场景** | 无线干扰、密钥不匹配、中间人攻击 |

#### 4. `payload-corrupt` - Payload损坏
| 属性 | 说明 |
|------|------|
| **触发条件** | random-30-percent (30%概率) |
| **行为** | 随机翻转Payload中的比特 |
| **网络反应** | 如果MIC正确则接受，但应用层数据损坏 |
| **检测方式** | 应用层CRC校验失败 |
| **实际场景** | 信道干扰、弱信号区域 |

#### 5. `mic-wrong-key` - MIC错误密钥 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 使用错误的NwkSKey计算MIC |
| **网络反应** | **MIC验证失败，数据包被丢弃** |
| **检测方式** | ChirpStack拒绝所有该设备的数据包 |
| **实际场景** | 密钥派生错误、会话密钥不匹配 |

#### 6. `wrong-devaddr` - 错误DevAddr (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 使用不属于该设备的DevAddr发送数据 |
| **网络反应** | 找不到设备记录，数据包被丢弃 |
| **检测方式** | ChirpStack日志显示未知DevAddr |
| **实际场景** | 设备克隆攻击、地址冲突 |

#### 7. `invalid-datarate` - 非法DataRate (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 使用当前区域不支持的DataRate（如DR15） |
| **网络反应** | 网关或NS可能拒绝 |
| **检测方式** | 检查`txpk`中的`datr`字段 |
| **实际场景** | 设备配置错误、区域参数不匹配 |

### 二、射频层异常 (6种)

#### 8. `signal-weak` - 弱信号
| 属性 | 说明 |
|------|------|
| **触发条件** | always |
| **行为** | RSSI=-145dBm, SNR=-25dB（极低信号） |
| **网络反应** | 可能接受但标记为弱信号，或丢包 |
| **检测方式** | 检查`rxpk`中的`rssi`/`lsnr`字段 |
| **实际场景** | 远距离传输、障碍物遮挡、天线故障 |

#### 9. `signal-spike` - 信号突变
| 属性 | 说明 |
|------|------|
| **触发条件** | random-30-percent |
| **行为** | RSSI在-30dBm到-130dBm之间随机跳动 |
| **网络反应** | 可能被接受，但ADR算法可能受影响 |
| **检测方式** | 监控RSSI异常波动 |
| **实际场景** | 多径效应、移动设备、干扰源 |

#### 10. `invalid-frequency` - 非法频率 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 使用当前区域未定义的频率（如868.1MHz在AS923区域） |
| **网络反应** | 网关可能过滤，NS可能拒绝 |
| **检测方式** | 检查`rxpk`中的`freq`字段 |
| **实际场景** | 区域配置错误、跨境部署 |

#### 11. `single-channel` - 单信道锁定 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 固定使用单一频率发送，忽略信道轮换 |
| **网络反应** | 可能接受，但频谱效率降低 |
| **检测方式** | 检查所有`rxpk`的`freq`是否相同 |
| **实际场景** | 设备固件bug、信道配置错误 |

#### 12. `duty-cycle-violation` - 占空比违规 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 发送间隔过短，超过区域占空比限制（如EU868的1%） |
| **网络反应** | 网关可能缓存或丢弃 |
| **检测方式** | 监控发送频率和占空比计算 |
| **实际场景** | 设备故障、合规性测试 |

#### 13. `adr-reject` - ADR指令拒绝 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 收到LinkADRReq后总是返回拒绝状态 |
| **网络反应** | NS无法优化该设备的数据速率 |
| **检测方式** | 检查MAC命令响应中的`LinkADRAns` Status |
| **实际场景** | 设备ADR实现bug、移动设备 |

### 三、行为层异常 (5种)

#### 14. `devnonce-repeat` - DevNonce重复
| 属性 | 说明 |
|------|------|
| **触发条件** | always |
| **行为** | 重复使用相同的DevNonce进行Join Request |
| **网络反应** | **Join Accept被拒绝**（重放攻击防护） |
| **检测方式** | ChirpStack日志显示"DevNonce already used" |
| **实际场景** | 设备复位后随机数生成器重置、攻击 |

#### 15. `rapid-join` - 快速Join
| 属性 | 说明 |
|------|------|
| **触发条件** | on-join-accept (Join成功后5秒) |
| **行为** | 收到Join Accept后5秒内立即重新发送Join Request |
| **网络反应** | 可能接受新的Join，原会话失效 |
| **检测方式** | 监控Join频率，检查`join`事件间隔 |
| **实际场景** | 设备Join逻辑bug、DoS攻击 |

#### 16. `burst-traffic` - 突发流量 (新增)
| 属性 | 说明 |
|------|------|
| **触发条件** | 配置触发 |
| **行为** | 短时间内发送大量数据包（如1秒内发送10个） |
| **网络反应** | 可能触发速率限制或丢弃 |
| **检测方式** | 监控`deduplication_id`时间戳密度 |
| **实际场景** | 设备故障、报警状态、DDoS攻击 |

#### 17. `random-drop` - 随机丢包
| 属性 | 说明 |
|------|------|
| **触发条件** | always (50%丢包率) |
| **行为** | 随机丢弃一半的数据包不上报 |
| **网络反应** | 数据不完整，confirmed包触发重传 |
| **检测方式** | 统计实际接收包数 vs 预期包数 |
| **实际场景** | 弱信号区域、网络拥塞 |

#### 18. `confirmed-noack` - Confirmed不确认
| 属性 | 说明 |
|------|------|
| **触发条件** | every-3rd-confirmed |
| **行为** | 发送Confirmed Uplink但忽略ACK，继续重传 |
| **网络反应** | 触发多次下行ACK，浪费资源 |
| **检测方式** | 监控重传次数和下行负载 |
| **实际场景** | 设备RX窗口故障、网络下行阻塞 |

---

### 触发类型支持

| 触发类型 | 说明 | 适用场景 |
|----------|------|----------|
| `always` | 每个包都触发 | 持续性异常 |
| `every-Nth-uplink` | 每第N个上行触发 | 间歇性异常 |
| `random-X-percent` | X%概率触发 | 随机异常 |
| `on-join-accept` | Join成功后触发 | 会话相关异常 |
| `once` | 仅触发一次 | 一次性测试 |
