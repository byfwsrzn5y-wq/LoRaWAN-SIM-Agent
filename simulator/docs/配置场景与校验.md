# LoRaSIM 配置场景、校验与 OpenClaw

**官方运行入口**：在 `simulator/` 目录执行 `node index.js -c <配置文件.json>`。  
对外产品与文档请以 **`index.js`** 为准（会展平 `version: "2.0"` 的 `simulation.gateway`）。`main.js` 已弃用，仅转发 `index.js`。

**对端网络服务器**：文档里常以 ChirpStack 为例；你可使用**自有或第三方的 ChirpStack（或其它兼容 Gateway Bridge 语义的环境）**，只要把模拟器 JSON 中的 `gatewayEui`、`lnsHost`/`lnsPort` 或 `mqtt.*`、密钥与对端**逐项对齐**即可，不强制使用本仓库内的 Docker 示例。

---

## 1. 场景 Profile（必填随场景而变）

| Profile ID | 用途 | 在校验命令中的 `--profile` |
|--------------|------|------------------------------|
| `v20-udp` | 经 Gateway Bridge 的 **UDP**（Semtech）连对端 NS（默认） | `v20-udp` |
| `mqtt` | 必须 `mqtt.enabled === true` | `mqtt` |
| `multigw` | 必须 `multiGateway.enabled` 且 `gateways[]` 非空 | `multigw` |
| `openclaw` | 与 `v20-udp` 同类；建议开 `controlServer` 供 Agent 调 `/reset` | `openclaw` |

各 Profile 的说明见源码 [`src/config/profiles.js`](../src/config/profiles.js) 中的 `PROFILE_META`。

### 1.1 三层标签（文档与 Schema `x-lorasim-tier`）

- **Required（scenario）**：在给定 Profile 下不配则报错或无法达到宣称行为（如 `multigw` 缺网关列表）。
- **Recommended**：不配有默认，但易与 ChirpStack 不一致（如 `lnsPort` 与 Bridge UDP 映射、MQTT `mqttTopicPrefix`）。
- **Optional / Advanced**：`signalModel`、`burst/jitter`、`stats` 等。

### 1.2 `v20-udp` / `openclaw` 最小语义（节选）

展平后（v2.0 会合并 `simulation.gateway`）通常需要：

- `gatewayEui`：16 位十六进制。
- `lnsHost` / `lnsPort`：指向 Gateway Bridge Semtech UDP。
- 若启用 `lorawan` 且启用上行：需有**设备来源之一**——`devices[]`、`lorawan.csvImportPath`、`lorawan.randomBehaviors` + 行为模板、`ABP` + `deviceCount`、`OTAA` 批量/单设备密钥等（与 `index.js` 加载分支一致）。缺省时校验器给出警告 **CS-CHK-DEV-SOURCE**。

## 2. 本地校验（人机与 CI 共用）

### 2.1 命令行

在**仓库根目录**（或按需改 `-c` / `--cwd`）：

```bash
node scripts/lorasim-config-validate.mjs -c simulator/config.json --profile v20-udp
```

在 `simulator/` 目录：

```bash
npm run validate-config
# 等价于: node ../scripts/lorasim-config-validate.mjs -c config.json --profile v20-udp
```

退出码：`0` 表示无 **errors**（warnings 仍可能存在）。

### 2.2 输出字段

- `errors` / `warnings`：每条可有 `checklistId`，与 [ChirpStack 联调配置核对清单](ChirpStack联调配置核对清单.md) 对齐。
- `normalizedPreview`：展平后的摘要（**不**包含完整 `appKey`）。

### 2.3 JSON Schema

机器可读字段定义：[`schemas/lorasim-config.schema.json`](../../schemas/lorasim-config.schema.json)（draft 2020-12）。

## 3. OpenClaw（AI Agent）

插件：[`openclaw-lorawan-sim`](../openclaw-lorawan-sim/)。

| 工具 | 作用 |
|------|------|
| `lorawan_sim_config_validate` | 与 CLI 相同规则；参数 `configPath`、`profile` |
| `lorawan_sim_config_set` | 可选 `validate_before_write: true`、`validate_profile`（默认 `openclaw`），校验失败**不写入** |

推荐流程：改配置前 `config_get` → 修改 → `config_validate` → `config_set`（带校验）→ `start`。
