# LoRaWAN 网关模拟器 — 功能清单

本文档基于当前代码与配置梳理**全部功能**，便于验收与排障参考。

---

## 一、代码与入口确认

| 项目 | 说明 |
|------|------|
| **主程序** | `index.js`（Node.js ≥14，单进程） |
| **启动方式** | `node index.js -c <config>` 或 `npm run start` / `start:config` / `start:100` / `start:otaa` |
| **配置目录** | `configs/`，主配置默认 `configs/config.json` |
| **OpenClaw 插件** | `openclaw-lorawan-sim/index.js`（ESM），通过 `api.registerTool` 注册工具 |

**依赖**：MQTT 模式需 `mqtt`、`protobufjs`（optionalDependencies）；UDP 模式仅需 Node.js。

## 二、核心模拟器功能

### 2.1 网关与 LNS 对接

| 功能 | 说明 |
|------|------|
| 网关身份 | 单一网关，`gatewayEui` 为 8 字节十六进制，须与 ChirpStack 网关 ID 一致 |
| UDP 模式 | 向 `lnsHost:lnsPort` 发送 PUSH_DATA / PULL_DATA，接收 PULL_RESP / PUSH_ACK / PULL_ACK |
| MQTT 模式 | 连接 Broker，发布 `{topicPrefix}/gateway/{gatewayId}/event/up`、`event/stats`，订阅 `command/down` |
| 协议版本 | Semtech Packet Forwarder Protocol Version 2 |
| 区域 | 支持 AS923-1 信道列表；可配 `uplink.region`、每设备 `nodeState.channels` |

### 2.2 上行（Uplink）

| 功能 | 说明 |
|------|------|
| 多设备调度 | 按配置/CSV/行为模板生成设备列表，每设备独立上行定时器 |
| 间隔与抖动 | `uplink.interval` / `intervalMs`，可选 `jitterRatio` 随机抖动 |
| 分散模式 | `scatterMode`: `random`（随机首包延迟）或 `uniform`（按设备数均分时间窗） |
| 负载 | **simple**（默认）：2–20 字节（`payloadLength`），前 2 字节为设备级递增计数器 + 随机填充；**custom**：`codec: "custom"` 时使用 `payload` + `payloadFormat`（hex/base64）作为 FRMPayload 明文，长度 0–222 字节。**多节点不同负载**：每设备可单独配置，通过 `config.devices[i].uplink` 或行为模板覆盖全局 uplink |
| 可选丢包 | `uplinkDropRatio`（0–1）模拟上行丢失 |
| 突发与静默 | `burstCount`、`burstIntervalMs`、`silenceAfterBurstMs` 控制突发后静默 |
| 首包后间隔 | `intervalAfterFirstDataMs` 用于首包后单独延迟（如 OTAA 首包后快速跟一包） |
| 信道选择 | 支持 `randomChannel`、区域信道、`macParams.channelMask` 约束、每设备 `nodeState.channels` |
| 发射功率/DR | 下行 MAC 可改 `macParams.dataRate`、`txPower`；上行据此选 SF、RSSI/SNR 偏移 |

### 2.3 下行（Downlink）

| 功能 | 说明 |
|------|------|
| 订阅 | MQTT 订阅 `{topicPrefix}/gateway/{gatewayId}/command/down` |
| 解析 | 支持 JSON 与 Protobuf（`mqtt.marshaler`）；解析 DownlinkFrame → phyPayload |
| Join Accept | MType=0x01：解密、派生 NwkSKey/AppSKey、写入设备会话、从 pending 移除、回 TxAck |
| 数据下行 | MType 0x03/0x05：解析 DevAddr、FCtrl、FCnt、FOpts（MAC）、FPort、FRMPayload；Confirmed 时置 needsAck |
| TxAck | 向 `event/ack` 发布 DownlinkTxAck（JSON 或 Protobuf） |

### 2.4 统计（Stats）

| 功能 | 说明 |
|------|------|
| 开关 | `stats.enabled`（默认 true）、`stats.intervalMs`（默认 60000） |
| 内容 | 发送 `stat` 对象（time, rxnb, rxok, rxfw, ackr, dwnb, txnb） |
| 输出 | UDP 模式：PUSH_DATA(stat)；MQTT 模式：发布 `event/stats`（JSON 或 Protobuf） |
| MQTT 上行追踪 | 订阅 `application/+/device/+/event/up`、`/rx`、`/log`，统计上行到达、延迟、FCnt 同步与 F_CNT_RESET 处理 |

## 三、LoRaWAN 协议

### 3.1 版本与激活

| 功能 | 说明 |
|------|------|
| 协议版本 | LoRaWAN 1.0.x（1.0.3 语义）：16-bit FCnt、OTAA Join、NwkSKey/AppSKey 派生 |
| ABP | 每设备 DevAddr、NwkSKey、AppSKey；支持 `devAddrStart` 递增、CSV 导入 |
| OTAA | Join Request（AppEUI、DevEUI、DevNonce、MIC）→ 等待 Join Accept → 派生密钥、写入会话；支持多设备、CSV、config.devices、randomBehaviors |

### 3.2 帧构建与加密

| 功能 | 说明 |
|------|------|
| 上行 ABP/已入网 OTAA | MHDR、FHDR（DevAddr、FCtrl、FCnt、FOpts）、FPort、加密 FRMPayload、MIC（NwkSKey） |
| 上行 FOpts | 携带 MAC 应答（LinkADRAns、DevStatusAns 等）及可选 LinkCheckReq；ACK 位回放下行确认 |
| FCnt | 每设备 16-bit 上行计数，内存态（预留持久化接口，当前未写盘） |
| 加密 | AES-128 ECB 用于 Join Accept；AES-CTR 风格（Ai 块 + AppSKey）用于 FRMPayload |

### 3.3 MAC 命令（下行解析与上行应答）

| CID | 名称 | 下行解析 | 上行应答/行为 |
|-----|------|----------|--------------|
| 0x02 | LinkCheckReq | — | 上行可选携带 LinkCheckReq（`linkCheckInterval`） |
| 0x02 | LinkCheckAns | — | — |
| 0x03 | LinkADRReq | DR/TxPower/ChMask/NbTrans | LinkADRAns；可配置 `adrReject` 拒绝；成功则更新 macParams |
| 0x04 | DutyCycleReq | — | DutyCycleAns（空） |
| 0x05 | RXParamSetupReq | RX1DROffset/RX2DR/RX2Freq | RXParamSetupAns；成功则更新 macParams |
| 0x06 | DevStatusReq | — | DevStatusAns（battery、margin）；可配置 `devStatus` |
| 0x07 | NewChannelReq | ChIndex/Freq/DRRange | NewChannelAns；成功则更新 customChannels |
| 0x08 | RXTimingSetupReq | — | RXTimingSetupAns（空） |
| 0x09 | TXParamSetupReq | MaxEIRP/DwellTime | TXParamSetupAns；更新 macParams |
| 0x0A | DLChannelReq | — | DLChannelAns(0x03) |
| 0x0D | DeviceTimeAns | — | （解析用） |

FPort=0 的 FRMPayload 按 NwkSKey 解密后同样解析为 MAC 命令并应答。

## 四、设备来源与行为

### 4.1 设备来源

| 来源 | 配置条件 | 说明 |
|------|----------|------|
| CSV 导入 | `lorawan.enabled` + `lorawan.csvImportPath` | 10 列：JoinMode,Group,Name,Profile,AppEUI,DevEUI,AppKey,DevAddr,AppSKey,NwkSKey；支持 ABP/OTAA 混合 |
| config.devices | `config.devices` 且含 OTAA | 每项 `lorawan.devEui/appEui/appKey`、可选 `nodeState`、`adrReject`、`devStatus`、`duplicateFirstData` |
| 随机行为模板 | `lorawan.randomBehaviors` + `deviceCount` + 模板文件/内联 | 从 `behaviorTemplatesFile` 或 `behaviorTemplates` 加载；按 `behaviorTemplateList` 或全部随机分配 |
| ABP 批量 | `lorawan.activation`=ABP + `deviceCount` | 按 devAddrStart/devEuiStart/nwkSKey/appSKey 生成 |
| OTAA 单/多节点 | `lorawan.appKey` + (appEui/devEui 或 appEuiStart/devEuiStart + deviceCount) | 单节点或递增 EUI 多节点 |

### 4.2 行为模板

| 功能 | 说明 |
|------|------|
| 模板结构 | `baseline` + `templates`；模板可 `extends: "baseline"` |
| 应用维度 | nodeState（channels、rssi、snr、txPowerIndex、rssiJitter、snrJitter）、uplink、lorawan、devStatus、adrReject、duplicateFirstData |
| nodeState 模式 | 显式配置或 `random`：rssiRange、snrRange、txPowerIndexRange、channelSubset/channelCount |
| 异常行为 | 见 `configs/behavior-templates.json` 与 docs：上行丢包、静默、ADR 拒绝、首包重复 FCnt 等 |

### 4.3 节点状态与 RF 模拟

| 功能 | 说明 |
|------|------|
| initNodeState | 每设备初始化 channels、txPowerIndex、rssi、snr、lastRssi/lastSnr、rssiJitter、snrJitter |
| 上行 RF | 按 nodeState 与 macParams 计算 RSSI/SNR（含抖动、TxPower 偏移）、SF（来自 dataRate）、信道池 |

## 五、控制与运维

### 5.1 HTTP 控制接口

| 功能 | 说明 |
|------|------|
| 开关 | `controlServer.enabled`、`controlServer.port`（默认 9999）、`controlServer.host` |
| 重置单设备 | GET/POST `/reset?devEui=...` 或 POST body `{ "devEui": "..." }`：OTAA 清会话（可 resetDevNonce），ABP 重置 FCnt |
| 重置全部 OTAA | GET/POST `/reset` 无 devEui：所有 OTAA 设备清会话 |
| 实现 | `index.js` 内 `resetDeviceByDevEui`、`resetAllOtaa`，供 HTTP 回调调用 |

### 5.2 FCnt 与 ChirpStack 同步

| 功能 | 说明 |
|------|------|
| 上行到达 | 订阅 application 的 event/up、rx，根据 fCnt 统计并调用 syncFCntFromChirpStack |
| F_CNT_RESET | 订阅 device log，收到 UPLINK_F_CNT_RESET 时 handleFCntReset，将对应设备 FCnt 置 0（预留，当前 updateFCntForDevice 为 no-op） |
| 持久化 | loadFCntState/saveFCntState/getFCntForDevice/updateFCntForDevice 已预留，当前为内存/空实现 |

### 5.3 其他

| 功能 | 说明 |
|------|------|
| CSV 导出 | 若配置 `lorawan.csvPath`（及可选 csvGroup、csvProfile），启动后导出当前 autoDevices 为 CSV |
| OTAA 定时重入网 | 每设备可配 `uplink.rejoinIntervalMs`，定时调用 resetLorawanDevice 触发重新 Join |
| 首包重复 FCnt | `duplicateFirstData`：首包后下一包使用相同 FCnt（模拟重复上行） |

## 六、OpenClaw 插件工具

### 6.1 模拟器工具（需 projectPath）

| 工具名 | 类型 | 说明 |
|--------|------|------|
| lorawan_sim_status | 只读 | 查看是否运行、PID、项目路径 |
| lorawan_sim_start | 可选 | 后台启动模拟器，可指定 configPath |
| lorawan_sim_stop | 可选 | 停止当前模拟器进程 |
| lorawan_sim_config_get | 只读 | 读取指定配置文件内容 |
| lorawan_sim_config_list | 只读 | 列出 configs 下所有 JSON 配置 |
| lorawan_sim_config_set | 可选 | 按 path/value 或 merge 对象更新配置 |
| lorawan_sim_reset_device | 可选 | 调用 controlServer /reset（需开启且模拟器运行） |
| lorawan_sim_sync_from_chirpstack | 可选 | 从 ChirpStack 拉取设备列表 → 生成 CSV → 更新 lorawan.csvImportPath 等 |

### 6.2 ChirpStack v4 工具（需 ChirpStack API 配置）

| 工具名 | 类型 | 说明 |
|--------|------|------|
| chirpstack_gateway_list | 默认 | 列出某租户网关（tenant_id 或 defaultTenantId） |
| chirpstack_gateway_create | 可选 | 注册网关（gateway_id 16 位 hex，name、description、tenant_id） |
| chirpstack_gateway_delete | 可选 | 删除网关 |
| chirpstack_device_list | 默认 | 列出某应用设备（application_id 或 defaultApplicationId） |
| chirpstack_device_create | 可选 | 创建设备 + CreateKeys（dev_eui、app_key 必填） |
| chirpstack_device_delete | 可选 | 删除设备 |
| chirpstack_downlink_send | 可选 | 下行入队（dev_eui、data；可选 f_port、confirmed） |

插件配置项：projectPath、chirpstackBaseUrl、chirpstackApiToken、chirpstackTenantId、chirpstackApplicationId、chirpstackDeviceProfileId（及对应环境变量）。

### 6.3 多节点不同负载与多 channel

- **多节点不同负载**：配置中需有 `config.devices`；通过 **lorawan_sim_uplink_payload_set** 的 **device_name** 或 **device_index** 可只改某一设备的上行负载；或用 **lorawan_sim_config_set** 的 merge 一次性写入多设备不同 `uplink`。
- **多 channel / 多 ChirpStack**：用**不同 configPath** 启动多个模拟器进程（如 `configs/gw1.json`、`configs/gw2.json`，各自不同 gatewayEui、MQTT），**lorawan_sim_start** 按 configPath 区分实例，**lorawan_sim_status** 不传参数列出全部，**lorawan_sim_stop** 传 configPath 停单实例、不传停全部。ChirpStack API 为单套；要对多套 ChirpStack 操作可配置多个插件条目。
