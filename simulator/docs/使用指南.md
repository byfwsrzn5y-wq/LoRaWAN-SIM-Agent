# 使用指南

从零到跑通「模拟器 + ChirpStack +（可选）OpenClaw Bot」的完整步骤。协议与 MAC 细节、行为模板机制见文档索引中的相关文档。

---

## 一、项目是什么、能做什么

- **LoRaWAN 网关模拟器**：在没有真实网关和终端时，用本程序模拟「一台网关 + 多台 LoRa 设备」，向 **ChirpStack** 发上行、收下行，用于联调、压力测试、行为测试。
- **协议**：LoRaWAN 1.0.3，支持 **OTAA** / **ABP**、MAC 命令（如 LinkADRReq、DevStatusReq 等）、simple 应用负载。
- **可选**：通过 **OpenClaw** 一个插件，用 Bot 管理模拟器（启停、改配置、设备重置）并在 **ChirpStack v4** 里管理网关/设备、发下行；还可把 ChirpStack 里的设备列表同步到模拟器，实现「ChirpStack 创建设备 → 模拟器用同一批设备发数据」。

**典型使用场景**：开发/联调上行解码与下行触发、100 节点正常+异常行为测试、通过对话完成网关/设备注册与模拟器启停。

---

## 二、环境与依赖

- **Node.js ≥ 14**
- **MQTT 模式**（与 ChirpStack Gateway Bridge 对接的推荐方式）：需安装 `mqtt`、`protobufjs`  
  ```bash
  npm install
  # 或显式：npm install mqtt protobufjs
  ```
- **UDP 模式**：不装 MQTT 也可用，仅需 Node.js（需 ChirpStack 支持 UDP Packet Forwarder）。

## 三、仅用模拟器（命令行）

不接 OpenClaw 时，在项目根目录用配置文件和命令行运行。

### 3.1 克隆与安装

```bash
git clone <repository-url> && cd lorawan_gateway_sim
npm install
```

### 3.2 准备配置

- 默认配置：`configs/config.json`（ABP、10 设备、MQTT、网关 EUI `0102030405060708`）。
- 复制一份按需修改（网关 EUI、MQTT 地址、设备数、OTAA/ABP 等）：
  ```bash
  cp configs/config.json configs/my-config.json
  # 编辑 configs/my-config.json
  ```

**必改项（与你的 ChirpStack 一致）**：

| 配置项 | 说明 |
|--------|------|
| `gatewayEui` | 网关 EUI，8 字节十六进制（如 `0102030405060708`）。ChirpStack 里创建的网关 ID 须与此一致。 |
| `mqtt.server` | MQTT Broker 地址（ChirpStack 使用的同一 Broker，如 `tcp://127.0.0.1:1883`）。 |
| `mqtt.topicPrefix` | 主题前缀（如 `as923`），与 ChirpStack Gateway Bridge 一致。 |
| `lorawan.*` | ABP 时：DevAddr、NwkSKey、AppSKey 与 ChirpStack 中设备一致；OTAA 时：AppEUI/DevEUI/AppKey 与 ChirpStack 中设备一致。 |

### 3.3 启动方式

```bash
# 使用默认 config.json
node index.js -c configs/config.json

# 使用自定义配置
node index.js -c configs/my-config.json

# 或使用 package.json 脚本
npm run start:config                    # configs/config.json
npm run start:100                       # 100 节点、10 种行为（OTAA + 行为模板）
npm run start:otaa                     # configs/config-otaa-demo-flow.json
```

### 3.4 ChirpStack 侧需要事先做好的事

1. 部署 **ChirpStack**（建议 v4），并确保 **Gateway Bridge** 使用 MQTT 时，与模拟器连同一 Broker。
2. 在 ChirpStack 中**创建网关**，Gateway ID 填配置里的 `gatewayEui`（如 `0102030405060708`）。
3. 创建**应用**与**设备**：  
   - **OTAA**：DevEUI、AppKey（及 AppEUI 若使用）与配置或 CSV 中一致。  
   - **ABP**：DevAddr、NwkSKey、AppSKey 与配置或 CSV 中一致。  
4. Gateway Bridge 订阅：`{topicPrefix}/gateway/{gatewayId}/event/+`，上行会发到同一 Broker。

这样模拟器启动后，会以「一台网关 + 多台设备」的身份发上行，ChirpStack 会收到并可下发下行。

### 3.5 预设配置说明

| 配置文件 | 用途 |
|----------|------|
| `configs/config.json` | 默认 ABP、10 设备。 |
| `configs/config-otaa-demo-flow.json` | OTAA 演示。 |
| `configs/config-otaa-mac-test.json` | OTAA + MAC 测试。 |
| `configs/config-100nodes-10types.json` | 100 节点、10 种行为（3 正常 + 7 异常）。 |
| `configs/config-100nodes-all-types.json` | 100 节点、全部行为模板。 |
| `configs/behavior-templates.json` | 行为模板定义（正常/异常）。 |

更多配置字段说明见根目录 [README.md](../README.md)#配置；行为模板与 100 节点见 [100 节点配置指南](100节点正常与异常配置.md)。

## 四、通过 OpenClaw Bot 使用（推荐完整流程）

用 **OpenClaw** 可通过对话完成：在 ChirpStack v4 注册网关/设备 → 把设备同步到模拟器 → 启停模拟器、改配置、发下行。只需安装**一个插件**：**openclaw-lorawan-sim**（已包含 ChirpStack v4 能力）。

### 4.1 安装 OpenClaw 与插件

```bash
# 安装 OpenClaw（二选一）
npm install -g openclaw
# 或中文版
npm install -g openclaw-cn@latest --registry=https://registry.npmmirror.com

openclaw onboard
```

在 OpenClaw 的配置中把**插件路径**指到项目根目录下的 `openclaw-lorawan-sim`（在 `openclaw.json` 或 `agents.json5` 的 `plugins.entries.lorawan_sim.path` 中填写本机绝对路径）。

### 4.2 ChirpStack v4 与 REST 代理

- 部署 **ChirpStack v4**。
- 运行 **chirpstack-rest-api**（将 gRPC 转成 REST），例如：  
  `--server localhost:8080 --bind 0.0.0.0:8090 --insecure`  
  插件会向该 REST 代理发请求（下面配置里的 `chirpstackBaseUrl` 即为此地址，如 `http://127.0.0.1:8090`）。

### 4.3 插件配置

在 OpenClaw 中为 `lorawan_sim` 配置至少如下项（也可用环境变量，见下表）：

| 配置项 | 说明 | 环境变量 |
|--------|------|----------|
| `projectPath` | 项目根目录的**绝对路径**（模拟器启停、读写配置必填） | `LORAWAN_SIM_PROJECT_PATH` |
| `chirpstackBaseUrl` | ChirpStack REST 代理地址 | `CHIRPSTACK_API_URL` |
| `chirpstackApiToken` | ChirpStack API 密钥 | `CHIRPSTACK_API_TOKEN` |
| `chirpstackApplicationId` | 默认应用 ID（列设备/创建设备/同步） | `CHIRPSTACK_APPLICATION_ID` |
| `chirpstackTenantId` | 默认租户 ID（创建网关） | `CHIRPSTACK_TENANT_ID` |
| `chirpstackDeviceProfileId` | 默认设备档案 ID（创建设备） | `CHIRPSTACK_DEVICE_PROFILE_ID` |

示例（仅示意结构）：

```json
{
  "plugins": {
    "entries": {
      "lorawan_sim": {
        "path": "/path/to/lorawan_gateway_sim/openclaw-lorawan-sim",
        "config": {
          "projectPath": "/path/to/lorawan_gateway_sim",
          "chirpstackBaseUrl": "http://127.0.0.1:8090",
          "chirpstackApiToken": "your-api-key",
          "chirpstackApplicationId": "your-application-uuid",
          "chirpstackTenantId": "your-tenant-uuid",
          "chirpstackDeviceProfileId": "your-device-profile-uuid"
        }
      }
    }
  }
}
```

在 agent 的 **tools.allow** 中启用需要用的可选工具（启动、停止、改配置、同步、创建设备、发下行等），例如：

```json
"allow": [
  "lorawan_sim_start", "lorawan_sim_stop", "lorawan_sim_config_set",
  "lorawan_sim_reset_device", "lorawan_sim_sync_from_chirpstack",
  "chirpstack_gateway_create", "chirpstack_gateway_delete",
  "chirpstack_device_create", "chirpstack_device_delete", "chirpstack_downlink_send"
]
```

### 4.4 推荐使用流程（ChirpStack → 同步 → 模拟器）

1. **在 ChirpStack v4 注册网关**  
   让 Bot 调用 `chirpstack_gateway_create`，gateway_id 填**与模拟器配置里一致的 8 字节十六进制**（如 `0102030405060708`），这样模拟器以该网关身份上行时 ChirpStack 能识别。

2. **在 ChirpStack v4 创建设备**  
   用 `chirpstack_device_create` 创建若干 **OTAA** 设备，**统一 AppKey**；可批量或逐个创建。

3. **同步到模拟器**  
   - 在模拟器配置中把 `lorawan.appKey` 设为同一 AppKey（或调用同步时传入 `app_key`）。  
   - 让 Bot 调用 **lorawan_sim_sync_from_chirpstack**：从 ChirpStack 拉取当前应用下的设备列表，生成 CSV 并更新模拟器配置，使模拟器用这批 DevEUI/AppKey 发数据。

4. **启动模拟器**  
   让 Bot 调用 **lorawan_sim_start**（可指定配置文件，如 `configs/config-100nodes-10types.json`）。模拟器会用已注册的网关 EUI 与同步来的设备做 OTAA 入网并持续发上行。

5. **后续操作**  
   - 查看状态：`lorawan_sim_status`  
   - 改配置：`lorawan_sim_config_set`（如上行间隔、设备数等）  
   - 停止：`lorawan_sim_stop`  
   - 发下行：`chirpstack_downlink_send`（指定 dev_eui、data 等）  
   - 设备重置（需先开 controlServer）：`lorawan_sim_reset_device`

这样「ChirpStack 里的网关 + 设备」与「模拟器使用的网关 + 节点」一致，便于端到端验证。

### 4.5 插件工具一览

- **模拟器**：`lorawan_sim_status`、`lorawan_sim_start`、`lorawan_sim_stop`、`lorawan_sim_config_get`、`lorawan_sim_config_list`、`lorawan_sim_config_set`、`lorawan_sim_reset_device`、`lorawan_sim_sync_from_chirpstack`
- **ChirpStack v4**：`chirpstack_gateway_list`、`chirpstack_gateway_create`、`chirpstack_gateway_delete`、`chirpstack_device_list`、`chirpstack_device_create`、`chirpstack_device_delete`、`chirpstack_downlink_send`

详细说明与参数见 [openclaw-lorawan-sim/README.md](../openclaw-lorawan-sim/README.md)。

## 五、设备重置与控制接口（可选）

若需要通过 HTTP 重置设备（例如 OTAA 重新入网、ABP FCnt 重置），需：

1. 在模拟器配置中开启控制接口，例如：  
   `"controlServer": { "enabled": true, "port": 9999 }`
2. 启动模拟器后，可：  
   - 用脚本或 curl 请求 `POST http://localhost:9999/reset`（可选 body 指定 devEui）；  
   - 或通过 Bot 调用 **lorawan_sim_reset_device**。

详见 [设备重置与重新入网](设备重置与重新入网.md)。

**常见问题**：

- **模拟器报未配置项目路径**：若用 OpenClaw，检查插件 `projectPath` 或环境变量 `LORAWAN_SIM_PROJECT_PATH` 是否为项目根目录绝对路径。
- **ChirpStack 工具报错**：检查 `chirpstackBaseUrl`、`chirpstackApiToken` 以及各默认 ID 是否正确；确认 chirpstack-rest-api 已启动且可访问。
- **上行 ChirpStack 收不到**：确认网关 ID 与 `gatewayEui` 一致、MQTT Broker 与 topic 一致、设备在 ChirpStack 已创建且密钥一致。
- **OTAA 入网失败**：确认 AppKey/DevEUI/AppEUI 与 ChirpStack 中设备完全一致；若刚同步设备，确认 AppKey 已写入配置或同步参数。

## 七、小结

| 使用方式 | 步骤概要 |
|----------|----------|
| **仅命令行** | 克隆 → `npm install` → 改 `configs/*.json`（gatewayEui、MQTT、lorawan）→ ChirpStack 建网关与设备 → `node index.js -c configs/xxx.json` |
| **OpenClaw 全流程** | 安装 OpenClaw → 配置 lorawan_sim 插件（projectPath + ChirpStack）→ 启用可选工具 → 对话：创建网关 → 创建设备 → 同步设备 → 启动模拟器 → 按需改配置/发下行/重置设备 |

按上述任一路径即可完整使用本项目进行 LoRaWAN 网关与设备模拟及 ChirpStack 联调。
