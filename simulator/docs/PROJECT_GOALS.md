# 项目目标与范围

## 目标

- **主目标**：在无真实 LoRa 网关与终端硬件时，用本模拟器 + ChirpStack 验证从「网关上行/下行」到「应用层设备管理」的完整链路。
- **扩展目标**：通过 OpenClaw Bot 完成模拟器启停、配置调整、ChirpStack v4 网关/设备注册与设备同步，便于自动化与联调。

## 范围

| 在范围内 | 不在范围内 |
|----------|------------|
| LoRaWAN 1.0.3 上行/下行、OTAA/ABP、MAC 命令 | LoRaWAN 1.1、Class B 时序、FUOTA |
| ChirpStack Gateway Bridge（MQTT/Protobuf 或 UDP） | 其他 NS/AS、私有协议 |
| 多节点、行为模板（正常/异常）、CSV 导入 | 私有应用层编解码、非 ChirpStack 协议栈 |
| OpenClaw 插件：模拟器管理 + ChirpStack v4 网关/设备/下行 | 其他 Bot 或自建控制台 |
| 设备重置、FCnt 同步、控制接口（HTTP） | 持久化 FCnt、完整统计 API |

## 使用场景

1. **开发/联调**：与 ChirpStack、应用服务联调上行解码、下行触发。
2. **压力与行为测试**：100 节点、多种行为模板（正常 + 异常）验证网络与策略。
3. **Bot 运维**：通过对话注册网关/设备、同步到模拟器、启停模拟器、下发下行。
