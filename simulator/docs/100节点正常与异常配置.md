# 100 节点配置指南

说明如何用现成预设或修改配置跑 **100 个节点**（正常 + 异常混合）。行为机制与模板定义见 [行为模型与随机节点](行为模型与随机节点.md)，异常模板含义见 [异常行为模板参考](异常行为模板参考.md)。

要模拟 **100 个不同节点**，其中既有**正常**行为也有**异常**行为，按下面步骤即可。

## 一、直接用现成配置（推荐）

项目里已有两套「100 节点」预设，二选一即可。

### 方式 A：10 种行为（约 3 正常 + 7 异常）

- **配置文件**：`configs/config-100nodes-10types.json`
- **含义**：100 个节点，每个节点**随机**从下面 10 种行为里选一种：
  - **正常**：`normal-near`、`normal-mid`、`normal-far`（近/中/远三种信号）
  - **异常**：`bad-snr-1`、`rejoin-5s`、`good-sf7-adr-fail`、`high-rate-confirmed`、`mac-heavy`、`lossy`、`channel-single`
- **启动**：
  ```bash
  node index.js -c configs/config-100nodes-10types.json
  ```
- **比例**：未指定比例时，每种行为被选中的概率相同，大致会得到约 30% 正常、70% 异常（因为 3 正常 / 7 异常）。

### 方式 B：全部行为类型（种类最多）

- **配置文件**：`configs/config-100nodes-all-types.json`
- **含义**：100 个节点，从 `configs/behavior-templates.json` 里的**全部**模板中随机选（含更多异常：低电量、首包重复、突发静默、SF 不合理、链路抖动等）。
- **启动**：
  ```bash
  node index.js -c configs/config-100nodes-all-types.json
  ```

## 二、从零「修复」你当前的 config.json

若你当前用的是默认 `configs/config.json`（ABP、10 设备），要改成「100 个 OTAA 节点 + 正常/异常混合」，需要改这几处：

| 原样（示例） | 改成 |
|--------------|------|
| `"activation": "ABP"` | `"activation": "OTAA"` |
| `"deviceCount": 10` | `"deviceCount": 100` |
| 无 | 增加 `"randomBehaviors": true` |
| 无 | 增加 `"behaviorTemplatesFile": "configs/behavior-templates.json"` |
| 无 | 增加 `"behaviorTemplateList": ["normal-near","normal-mid","normal-far", ...]` 或 `"all"` |
| ABP 的 nwkSKey/appSKey/devAddrStart | 改为 OTAA 的 `appEuiStart`、`devEuiStart`、`appKey`（见下方示例） |

**最小改动示例**（在 `config.json` 的 `lorawan` 里替换/合并为）：

```json
"lorawan": {
  "enabled": true,
  "activation": "OTAA",
  "class": "C",
  "appEuiStart": "0000000000000001",
  "devEuiStart": "0102030405060701",
  "appKey": "00112233445566778899AABBCCDDEEFF",
  "randomBehaviors": true,
  "deviceCount": 100,
  "behaviorTemplatesFile": "configs/behavior-templates.json",
  "behaviorTemplateList": [
    "normal-near",
    "normal-mid",
    "normal-far",
    "bad-snr-1",
    "rejoin-5s",
    "good-sf7-adr-fail",
    "high-rate-confirmed",
    "mac-heavy",
    "lossy",
    "channel-single"
  ]
}
```

这样就是 100 个 OTAA 节点，从这 10 种行为里随机分配。若要更多异常类型，把 `behaviorTemplateList` 改成 `"all"`，或从 `behavior-templates.json` 的 `templates` 里挑更多 ID 放进数组。

## 三、调整「正常 vs 异常」比例

- **behaviorTemplateList** 是「允许被抽到的行为类型」列表，每个节点**等概率**从列表中随机选一种。
- 想让正常多一点：在数组里多写几次正常模板，例如：
  ```json
  "behaviorTemplateList": [
    "normal-near", "normal-near", "normal-mid", "normal-mid", "normal-far",
    "bad-snr-1", "rejoin-5s", "lossy", "channel-single"
  ]
  ```
  这样 5/9 是正常，4/9 是异常。
- 想用上全部已定义行为：设 `"behaviorTemplateList": "all"`（如 `config-100nodes-all-types.json`）。

## 四、ChirpStack 侧必须配合

- 模拟器用 **OTAA**，且 `deviceCount: 100`、`devEuiStart: "0102030405060701"` 时，会生成 100 个 DevEUI：  
  `0102030405060701`、`0102030405060702`、…、`0102030405060764`（十六进制递增）。
- 在 ChirpStack 里需要：
  1. 创建**同一应用**下的 **100 个 OTAA 设备**；
  2. 每个设备的 **DevEUI** 与上面范围一致；
  3. **AppKey** 与配置里的 `lorawan.appKey` 一致（如 `00112233445566778899AABBCCDDEEFF`）。
- 若不做这一步，模拟器发出的 Join Request 在 ChirpStack 侧没有对应设备，无法入网。

## 五、行为模板说明

- 所有行为（正常/异常）定义在 **`configs/behavior-templates.json`**。
- **baseline**：默认「正常」行为；各模板可用 `"extends": "baseline"` 只写与正常行为的差异。
- 常见异常含义简表：

| 模板 ID | 含义 |
|---------|------|
| normal-near / normal-mid / normal-far | 正常，仅距离/信号不同 |
| bad-snr-1/2/3 | 信号差、SNR 差、抖动大 |
| rejoin-5s | 频繁重入网 |
| good-sf7-adr-fail / sf12-good-signal / sf7-poor-link | ADR 不合理、SF 固定 |
| high-rate-confirmed | 高频率 + confirmed |
| mac-heavy | 以 MAC 为主、应用数据少 |
| lossy / confirmed-lossy | 上行丢包、确认包重传多 |
| low-battery | 低电量/差裕量上报 |
| join-then-sleep | 入网后发一包即长睡 |
| duplicate-fcnt | 首包 FCnt 重复 |
| burst-silence | 突发后长静默 |
| channel-single | 单信道 |
| link-jittery | 链路波动大 |

更多细节见 [行为模型与随机节点](行为模型与随机节点.md) 与 [异常行为模板参考](异常行为模板参考.md)。

## 六、小结

- **要 100 个节点、有正常有异常**：用 `configs/config-100nodes-10types.json` 或 `configs/config-100nodes-all-types.json`，再在 ChirpStack 里配好 100 个 OTAA 设备即可。
- **要基于现有 config.json 改**：把 `lorawan` 改成 OTAA + `randomBehaviors: true` + `deviceCount: 100` + `behaviorTemplatesFile` + `behaviorTemplateList`（数组或 `"all"`）。
- **调比例**：在 `behaviorTemplateList` 数组里多放或少放某种行为 ID；用 `"all"` 则种类最多、比例均等。
